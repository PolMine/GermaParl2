5.1.2 External Data

The majority of the information added to the initial protocols originates from the Stamdaten of the German Bundestag. It comprises information about Members of Parliament. Party affiliations are added from Wikipedia. In the preparation process, the Stammdaten file was converted into a data.table object which was then stored in a R data package for versioning and documentation purposes. Other speakers are enriched with Wikipedia. In cases in which no information about a speaker could be found on Wikipedia, Munzinger proofed a valuable resource for speaker names and party affiliations.

5.2 Tools

Stanford CoreNLP

For the current iteration of the corpus, the Java version of Stanford CoreNLP (version 4.5.x) was used to perform the initial linguistic annotation (Manning et al. 2014). More specifically, tokenization, splitting of sentences, Part-of-Speech tagging in the Universal Dependencies tag set and Named Entity Recognition were performed using the default German language model. To make use of the parallel computing capabilities of Stanford CoreNLP from within R, the R wrapper bignlp was developed in the context of the PolMine project (BlÃ¤tte and Leonhardt 2022). It is available on GitHub.

TreeTagger

To add lemmata and language specific Part-of-Speech tags to the current corpus, TreeTagger was used (Schmid 1994). While not the most recent solution to add these annotation layers, TreeTagger is fast and robust.

Corpus Workbench

The corpus is also provided in the format of the the IMS Corpus Workbench.. The preparation workflow mainly communicates with the CWB via the cwbtools R package (BlÃ¤tte 2022a) which is used for the encoding of the data.

Additional R Packages

The workflow is set up in R (R Core Team 2021). For different parsers, the packages xml2 (Wickham, Hester, and Ooms 2021) and stringr (Wickham 2022) provide important functionality. To facilitate the structural annotation of the protocols, the R package frappp is crucial. The R package trickypdf (BlÃ¤tte and Leonhardt 2019) was used to resolve the two-column layout of PDF files.
